# engagement-quality-gate

**Category:** Engagement  
**Trigger:** Work is complete but not yet delivered — final review before client delivery

---

## What It Does

The final checkpoint before any deliverable reaches a client or stakeholder. Runs a structured quality review across five dimensions (accuracy, clarity, audience specificity, strategic voice, format), checks that every scoped item was delivered, verifies objective alignment, and flags anything that must be resolved before sending. Then produces a polished case study capturing the work and value delivered — usable for internal knowledge capture, portfolio, or business development.

---

## When to Use

- Work is "done" but hasn't gone to the client yet
- You want a structured QA pass before a high-stakes delivery
- You need to capture the engagement story for your portfolio or BD pipeline
- A deliverable needs a second set of eyes on completeness and alignment

---

## Trigger Phrases

- "We're ready to send this"
- "Review this before it goes out"
- "QA this before delivery"
- "Is this complete?"
- "Ready to wrap this up"

---

## Key Inputs

| Input | Required? | Notes |
|---|---|---|
| The deliverable(s) | ✅ | File, report, deck, tool, briefing |
| Original objectives | ✅ | Pull from planning conversation or ask |
| Client / Recipient | ✅ | Who receives this |
| Scope commitments | Optional | Sharpens completeness check |
| Success criteria | Optional | Sharpens objective alignment |

---

## Outputs

**Quality Review Report:**
- Scope Coverage Matrix (✅ / ⚠️ / ❌ per item)
- Five-dimension quality assessment
- Objective alignment table
- Gap resolution plan with severity levels
- Delivery Decision: 🟢 CLEARED / 🟡 CONDITIONAL / 🔴 HOLD

**Engagement Case Study:**
- Situation, objective, approach, deliverables, value delivered
- 400–700 words
- Tagged for internal search and portfolio use
